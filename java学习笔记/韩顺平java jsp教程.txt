## servlet基础入门

#### web开发简介

* HTML产生于1993年，属于静态网页，不能做动态的网页

* 要做动态网页，必须要用到服务器

* ASP不是一门语言，而是一个运行环境 ，他是基于HTML JavaScript vb 的

* 它的服务器是，IIS+window，效率比CGI高

* 由于ASP是基于window和IIS的，所以相应的漏洞也就比较多，它的跨平台不是很好，一般认为，只能在window系统上面运行

* ASP只能作为中小型项目，大型门户网站比较吃力，大型门户网站一般用.net 或者php做后台的

* PHP的优势：跨平台良好，效率高，安全，免费
不足：安装复杂，由zend公司负责管理

* PHP+mysql+Linux+Apache全部都是免费，而且效率也是不错的

**韩老师是讲完jsp再讲PHP的！！！**

#### jsp的优势：

- 一次编写，到处运行，良好的跨平台型，也就是三大平台 window Linux unix

- 强大的可伸缩性，jsp+javabean

#### b/s 与 c/s 比较

这两者本质上没有什么区别

对于c/s 服务器和客户端都是我们自己开发的

b/s 的客户端就是浏览器，我们只需要开发服务器端

所以b/s的开发成本比较低，只需要管服务器端，不用管浏览器端，产品升级便利

如果是c/s架构的，必须先升级服务器端，才能升级客户端

c:client
b:browser

servlet介绍：
就是用java语言编写的服务器端程序

servelet规范：
只要是开发servelet就必须遵循servlet规范

安装tomcat产生的问题？500错误

问题解决：原来真的是tomcat版本和jdk版本不一致造成的，必须同样是64位才可以正常运行

首先，需要考虑到是不是安装的tomcat版本和自己安装的jdk的版本一致，如果不一致，建议将tomcat换成和jdk一致或者高于jdk版本的tomcat版本。（比如安装的jdk是1.7，那么tomcat的版本一定要不低于1.7的版本）

其次，需要考虑安装的jdk和tomcat是不是都是32位的或者都是64位的，如果不是建议安装同样系统版本的版本。
